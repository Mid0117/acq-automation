The Frame

Three crons run on Microsoft's servers (free, public repo). They handle every step that used to need a human: transcribing calls, scoring leads, building rehab reports, sending follow-up SMS, classifying replies, advancing stages, and pricing properties. Two live dashboards — Follow-Ups and Deals — show what's happening and what needs attention.

Net result. Jeff calls people who actually want to talk. Mike reviews instead of doing data entry. Cold leads stop clogging the qualified queue. ARV gets calculated automatically. The team only sees what needs a human decision.

DESIGN RULE

Trust the AI for routing, never for closing. Calls get analyzed, leads get classified, stages get auto-routed when the system is confident — but a deal never moves from LAO → DD → MAO → Contract without a human signing off. The automation handles the noise so humans handle the signal.

Call Recording Pipeline

EVERY 15 MIN acq_automation.py

"Find new recordings. Transcribe them. Extract everything that matters. Route the lead. Move on."

Quick model

Recording-driven enrichment. Every 15 min, the cron checks GHL for new call recordings on stages 1-4. Each one runs through Deepgram → Claude → Apify → GHL writeback in under a minute. Side effect: the lead gets its stage auto-routed based on the call's outcome, and a clean "APG Lead Summary" note replaces the old freeform clutter.

Per-recording cost
~$0.04
Deepgram transcription + Claude analysis + Apify enrichment, all-in.
Time recovered
~30 min/call
Mike's old job: listen, type, look up Zillow, write summary, set fields.
Coverage
100%
Every recording analyzed. Old workflow caught maybe 10-20%.
What flows IN
  • Call recording URL from GHL
  • Contact's existing custom fields
  • Apify Zillow detail (beds/baths/sqft/year)
  • Apify sold comps in the same ZIP
What flows OUT
  • Transcript stored on contact
  • 16 GHL custom fields populated
  • "APG Lead Summary" note (upserted)
  • Rehab Report Google Doc
  • Auto-routed stage + Lead Type
  • Tasks for Jeff/Adam/Mike (if Hot)

Auto-routing logic

  • Hot → moves Stage 1 → Stage 2 (LAO), creates Jeff/Adam/Mike tasks (4h due), sets Lead Type = "Hot Lead".
  • Warm → stays in Stage 1, sets Lead Type = "Warm Lead". Still in qualified queue.
  • Nurture → moves to Stage 0 (Unqualified), sets Lead Type = "Nurture Lead". Silent demotion.
  • Cold → moves to Stage 0 (Unqualified), sets Lead Type = "Cold Lead". Silent demotion.

Stages 2-4 are never auto-reversed. Once a lead is a real deal, only humans move it.

02

SMS Follow-Up + Dashboards

EVERY 30 MIN sms_followup.py + 3 dashboards

"The right SMS, at the right time, on the right number. Filter the noise. Push the signal to Jeff."

Quick model

Cadence-driven outbound. The cron walks every active lead in stages 1-4 + the three reactivation stages, checks where they are in the SMS sequence, and sends the next message if it's due. Inbound replies get classified by Claude before Jeff sees anything — STOP/hostile/wrong-number all auto-DND silently. Only positive/neutral replies reach Jeff's task list.

Hours/week saved
25-40h
Jeff + Mike's combined manual SMS + reply triage time.
From-numbers
9
8 state-routed primaries + Jeff NJ. Secondary swap on touches 4-6.
Reply latency
≤ 30 min
Bounded by cron tick. Webhooks would push this to seconds.
Before — manual
  • SMS sent by hand, missed days, typos
  • Replies trickle in, Jeff scrambles
  • STOP requests sometimes ignored (legal risk)
  • No record of which from-number was used
After — automated
  • Right SMS, right time, right state-routed number
  • Replies classified before Jeff sees them
  • "Reply STOP to opt out" appended on touches 1, 4, last
  • Negative replies auto-DND, never escalated

Reply classifier (Claude Haiku)

  • STOP / hostile → set GHL DND, tag dnd-opt-out, no Jeff task.
  • Wrong number → set GHL DND, tag wrong-number, no task.
  • Not interested (polite no) → tag not-interested, no Jeff task. Re-touch eligible later.
  • Positive / neutral → Jeff callback task + Mike review task, both with reply text inline.

Dashboards (also produced here)

  • Google Sheet — raw rows, kill-switch toggle, live-editable templates.
  • Follow-Ups HTML — action queue: Replied / Dormant / Active.
  • Deals HTML — property cards: ARV, 70% MAO, assignment fee, call rating.

Slack Scraper

EVERY 60 MIN slack_scraper.py

"Watch the team chatter. When someone mentions a lead, capture it before it's lost in the channel."

Quick model

Channel-driven enrichment. Every hour, the cron pulls new messages from 4 APG Slack channels, locally scores them against the contact index (no LLM call until there's a candidate hit — keeps cost down), then sends matched candidates to Claude for confidence-rated info extraction. High-confidence updates auto-apply to GHL fields; medium-confidence becomes an audit-trail note.

Channels watched
4
#base0-hot-warm-nurture-fu, #base1-sms-leadgen, #base4-dispo, #construction-services
Local pre-filter
~98%
Of messages skip Claude entirely (no candidate match). Keeps cost per match low.

High only
Medium gets a note. Low/none gets nothing. No silent overwrites.

COST GUARDRAILS

The Apify budget guard checks remaining monthly limit before every call. Once we're $1 from the cap, all enrichment stops cleanly — no surprise overruns. Claude usage scales linearly with call volume; predictable. Deepgram is pay-as-you-go so it tracks call minutes 1:1.

For context: a single hour of Jeff's time replaced by automation pays for the entire system for a month.

Roadmap

▸ Now Building

A/B template testing In Progress

Tag every send with which template variant fired. Track reply rate per variant in the dashboard. After ~200 sends per variant we'll know which copy converts. Foundation for ML-driven copy optimization later.

Predictive lead scoring In Progress

Score every new lead 0-100 before the first call using property + motivation + timeline + market signals. Jeff prioritizes by score. Heuristic version ships now; ML-trained version once APG hits ~50 closed deals as training data.

Cold lead reactivation In Progress

Properties that went Dead 6+ months ago — life events change. Re-touch sequence with property-specific Zestimate updates ("market's up 8% in your area, want a new number?"). Builds on the existing Follow Up cadence with smarter copy.

Auto rehab scope from photos In Progress

Pull listing photos via Apify, send to Claude vision → automatic repair scope per room + a rough $/sqft repair estimate. Replaces ~30 min of manual estimator time per property and feeds straight into the existing Rehab Report doc.

▸ Future

Real-time reply detection Medium

Switch the SMS reply path from 30-min polling to a GHL webhook for sub-second detection. Means Jeff can call back while the seller is still thinking about it.

Calendly auto-booking Medium

When a positive reply comes in, parse the seller's stated availability. Send back a Calendly link pre-filtered to matching slots. Saves Jeff a round-trip.

Multi-language SMS Future

Detect seller's preferred language. Maintain Spanish + Portuguese template tracks for relevant markets (NJ has heavy Hispanic + Brazilian populations).

Voice AI cold caller Future

For "Call Needed" leads with no recording yet — AI voice agent makes the first qualifying call. Only escalates to Jeff if Hot. Tackles the biggest remaining bottleneck.

Cash buyer dashboard Future

Mirror the ACQ deals dashboard for the Cash Buyer pipeline (OH/PA/NJ/AL). Track buyer engagement, last contact, capacity per buyer.

Spend-per-deal analytics Future

Break down Apify + Claude + Deepgram spend per lead. Calculate APG's true cost-per-closed-deal. Tells us which lead sources are profitable.

Construction services pipeline Future

Same call → transcript → analyze flow but for the Construction Services pipeline. Different prompt, different fields, different routing — same infrastructure.
