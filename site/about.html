The 30-Second Version

Three crons run on Microsoft's servers (free, public repo). Together they handle every step that used to need a human: transcribing calls, scoring leads, building rehab reports, sending follow-up SMS, classifying replies, advancing stages, and pricing properties. Two live dashboards show you what's happening and what needs human attention.

Net result: Jeff calls people who actually want to talk. Mike reviews instead of doing data entry. Cold leads stop clogging the qualified queue. ARV gets calculated automatically. You get notified only when a human decision is required.

The Three Workers running 24/7

📞 Call Recording Pipeline
Every 15 min
acq_automation.py
  • Finds new call recordings in stages 1-4 of the ACQ pipeline.
  • Sends audio to Deepgram → transcript.
  • Sends transcript to Claude → extracts beds/baths/sqft, condition, asking price, motivation, timeline, deal type, lead temp (Hot/Warm/Nurture/Cold), call rating /10, action items, red flags.
  • Calls Apify Zillow → property details + sold comps in the same ZIP.
  • Sends comps to Claude → ARV estimate. Falls back to Zestimate if comps thin.
  • Writes everything back to GHL: contact custom fields, opportunity custom fields, "APG Lead Summary" note (single upserted note — no duplicates), Rehab Report Google Doc.
  • Auto-routes Stage 1 leads: Hot → LAO + tasks for Jeff/Adam/Mike. Warm stays. Nurture/Cold → Unqualified. Lead Type field set on every move.
  • Backfill phase: any Stage 1-4 lead missing ARV gets Apify-enriched (capped 15/run, ZIP-cached so clusters share one fetch).
💬 SMS + Dashboards
Every 30 min
sms_followup.py + dashboard.py + dashboard_html.py + deals_dashboard.py
  • Reads kill switch + live templates from Google Sheet (Settings tab + Templates tab — edit copy without a code push).
  • For every active lead in stages 1-4 + Follow Up 1.5mo / 3mo / Dead Deals: sends next SMS in sequence based on per-stage cadence.
  • From-number routing by state: AL/GA/IN/OH/PA/SC/TN/WI each have a primary number; touches 4-6 swap to a secondary so you don't look like one spammer.
  • TCPA compliance: "Reply STOP to opt out." appended on touches 1, 4, and the final touch.
  • Reply detection + classification: when an inbound SMS arrives, Claude classifies it: STOP / hostile / wrong number / not interested / positive / neutral. Negative replies set GHL DND silently — no Jeff task. Positive/neutral creates the callback pack.
  • Dormant detection: after 6 unanswered SMS, mark dormant + manual-call task for Jeff + review task for Mike.
  • Refreshes 3 dashboards: Google Sheet (raw data), Follow-Ups HTML (action queue), Deals HTML (deal cards with ARV / call rating / assignment fee).
  • Deploys the HTML pages to GitHub Pages.
📡 Slack Scraper
Every 60 min
slack_scraper.py
  • Pulls new messages from #base0-hot-warm-nurture-fu, #base1-sms-leadgen, #base4-dispo, #construction-services.
  • Local keyword/phone matching against the contact index (no LLM call until there's a candidate).
  • For matched candidates: Claude extracts info — match confidence, summary, optional field updates (asking price, timeline, motivation, reason for selling).
  • Adds an audit-trail "Slack mention" note to the GHL contact. Auto-applies field updates only at HIGH confidence.
  • Respects Slack's May 2025 rate limits (limit=15, sleep between pages).

Cost guardrails baked in

The Apify budget guard checks remaining monthly limit before every call. Once you're $1 from the cap, all enrichment stops cleanly — no surprise overruns. Claude usage is per-event so it scales linearly with call volume; if calls double, Claude cost doubles, predictably. Deepgram is pay-as-you-go so it tracks call minutes 1:1.

What to Add Next prioritized

Real-time reply detection High

Today: replies are detected on the next 30-min cron tick. Switch to a GHL webhook for sub-second detection. Means Jeff can call back while the seller is still thinking about it. Hosting: Cloudflare Workers free tier.

A/B template testing High

Tag every send with which template variant fired. Track reply rate per variant in the dashboard. After ~200 sends per variant you'll know which copy converts. Foundation for ML-driven copy optimization later.

Calendly auto-booking High

When a positive reply comes in, parse the seller's stated availability (Claude already does this). Send back a Calendly link pre-filtered to matching slots. Saves Jeff an entire round-trip of "what time works for you?"

Predictive lead scoring Medium

Train on closed deals: which property + motivation + timeline combos actually close. Score every new lead 0-100 before the first call. Jeff prioritizes by score. Becomes possible once we have ~50 closed deals as training data.

Dispo auto-matching Medium

When a deal hits Contract stage, match the property against your cash buyer pipeline (price range, area, type) and auto-draft an introduction email + dispatch list. Closes the loop into the dispo workflow.

Cold lead reactivation Medium

Properties that went Dead 6+ months ago — life events change. Re-touch sequence with property-specific Zestimate updates ("market's up 8% in your area, want a new number?"). Currently stage exists but no automation drives it.

Auto rehab scope from photos Medium

Pull listing photos via Apify, send to Claude vision → automatic repair scope per room. Replaces ~30 min of manual estimator time per property.

Multi-language SMS Future

Detect seller's preferred language from inbound replies. Maintain Spanish + Portuguese template tracks. Opens markets where current outreach is invisible.

Voice AI cold caller Future

For Stage 0 / "Call Needed" leads with no recording yet — AI voice agent makes the first qualifying call, transcript flows into the same pipeline. Only escalates to Jeff if Hot. Currently the most expensive bottleneck (Jeff time) has no automation.

Cash buyer dashboard Future

Mirror the ACQ deals dashboard but for the Cash Buyer pipeline (OH/PA/NJ/AL). Track buyer engagement, last contact, capacity per buyer.

Spend analytics page Future

Break down Apify + Claude + Deepgram spend by lead. Calculate cost-per-closed-deal. Helps decide which lead sources are actually profitable.

Construction services pipeline Future

Same call → transcript → analyze flow but for the Construction Services pipeline. Different prompt, different fields, different routing — but identical infrastructure.
