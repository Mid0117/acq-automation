Three Robots, Working 24/7

Built specifically for APG's acquisitions workflow. Every call gets transcribed, every lead gets scored, every follow-up SMS goes out on time, every reply gets classified, every property gets priced. The team only sees what needs a human decision — Jeff calls people who actually want to talk; Mike reviews instead of doing data entry.

The 30-Second Version

Three crons run on Microsoft's servers (free, public repo). Together they handle every step that used to need a human: transcribing calls, scoring leads, building rehab reports, sending follow-up SMS, classifying replies, advancing stages, and pricing properties. Two live dashboards (Follow-Ups and Deals) show what's happening and what needs attention.

The Three Workers running 24/7

📞 Call Recording Pipeline
Every 15 min
acq_automation.py
  • Finds new call recordings in stages 1-4 of the APG ACQ pipeline.
  • Sends audio to Deepgram → transcript.
  • Sends transcript to Claude → extracts beds/baths/sqft, condition, asking price, motivation, timeline, deal type, lead temp (Hot/Warm/Nurture/Cold), call rating /10, action items, red flags.
  • Calls Apify Zillow → property details + sold comps in the same ZIP.
  • Sends comps to Claude → ARV estimate. Falls back to Zestimate if comps thin.
  • Writes everything back to GHL: contact + opportunity custom fields, "APG Lead Summary" note (single upserted note — no duplicates), Rehab Report Google Doc.
  • Auto-routes Stage 1 leads: Hot → LAO + tasks for Jeff/Adam/Mike. Warm stays. Nurture/Cold → Unqualified. Lead Type field set on every move.
  • Backfill phase: any Stage 1-4 lead missing ARV gets Apify-enriched (capped 15/run, ZIP-cached so clusters share one fetch).
💬 SMS + Dashboards
Every 30 min
sms_followup.py + 3 dashboard scripts
  • Reads kill switch + live templates from APG's Google Sheet (Settings + Templates tabs — Mike can edit copy without a code push).
  • For every active lead in stages 1-4 + Follow Up 1.5mo / 3mo / Dead Deals: sends next SMS in sequence based on per-stage cadence.
  • From-number routing by state: 8 state-specific numbers + Jeff's NJ line; touches 4-6 swap to a secondary state's number to avoid spam-flagging.
  • TCPA compliance: "Reply STOP to opt out." appended on touches 1, 4, and the final touch.
  • Reply detection + classification: when an inbound SMS arrives, Claude classifies it: STOP / hostile / wrong number / not interested / positive / neutral. Negative replies set GHL DND silently — no Jeff task. Positive/neutral creates the callback pack for Jeff with Mike review.
  • Dormant detection: after 6 unanswered SMS, mark dormant + manual-call task for Jeff + review task for Mike.
  • Refreshes 3 dashboards: Google Sheet (raw data), Follow-Ups HTML (action queue), Deals HTML (deal cards with ARV / call rating / assignment fee).
📡 Slack Scraper
Every 60 min
slack_scraper.py
  • Pulls new messages from #base0-hot-warm-nurture-fu, #base1-sms-leadgen, #base4-dispo, #construction-services.
  • Local keyword/phone matching against the contact index (no LLM call until there's a candidate — keeps cost down).
  • For matched candidates: Claude extracts info — match confidence, summary, optional field updates (asking price, timeline, motivation, reason for selling).
  • Adds an audit-trail "Slack mention" note to the GHL contact. Auto-applies field updates only at HIGH confidence.
  • Respects Slack's May 2025 rate limits (limit=15, sleep between pages).

Cost guardrails baked in

The Apify budget guard checks remaining monthly limit before every call. Once we're $1 from the cap, all enrichment stops cleanly — no surprise overruns. Claude usage scales linearly with call volume; if calls double, Claude cost doubles, predictably. Deepgram is pay-as-you-go so it tracks call minutes 1:1.

For context: a single hour of Jeff's time replaced by automation pays for the entire system for a month.

Roadmap

▸ Now Building

A/B template testing In Progress

Tag every send with which template variant fired. Track reply rate per variant in the dashboard. After ~200 sends per variant we'll know which copy converts. Foundation for ML-driven copy optimization later.

Predictive lead scoring In Progress

Score every new lead 0-100 before the first call using property + motivation + timeline + market signals. Jeff prioritizes by score. Initial heuristic-based version ships now; ML-trained version once APG hits ~50 closed deals as training data.

Cold lead reactivation In Progress

Properties that went Dead 6+ months ago — life events change. Re-touch sequence with property-specific Zestimate updates ("market's up 8% in your area, want a new number?"). Builds on the existing Follow Up 1.5mo / 3mo / Dead Deals cadence with smarter copy.

Auto rehab scope from photos In Progress

Pull listing photos via Apify, send to Claude vision → automatic repair scope per room + a rough $/sqft repair estimate. Replaces ~30 min of manual estimator time per property and feeds straight into the existing Rehab Report doc.

▸ Future

Real-time reply detection Medium

Switch the SMS reply path from 30-min polling to a GHL webhook for sub-second detection. Means Jeff can call back while the seller is still thinking about it. Hosting: Cloudflare Workers free tier.

Calendly auto-booking Medium

When a positive reply comes in, parse the seller's stated availability (Claude already does this). Send back a Calendly link pre-filtered to matching slots. Saves Jeff an entire round-trip of "what time works for you?"

Multi-language SMS Future

Detect seller's preferred language from inbound replies. Maintain Spanish + Portuguese template tracks for relevant markets (NJ has heavy Hispanic + Brazilian populations). Opens markets where current outreach is invisible.

Voice AI cold caller Future

For "Call Needed" leads with no recording yet — AI voice agent makes the first qualifying call, transcript flows into the same pipeline. Only escalates to Jeff if Hot. Currently the most expensive bottleneck (Jeff time) has no automation.

Cash buyer dashboard Future

Mirror the ACQ deals dashboard but for the Cash Buyer pipeline (OH/PA/NJ/AL). Track buyer engagement, last contact, capacity per buyer, match rate vs APG dispo flow.

Spend-per-deal analytics Future

Break down Apify + Claude + Deepgram spend per lead. Calculate APG's true cost-per-closed-deal. Helps decide which lead sources are actually profitable.

Construction services pipeline Future

Same call → transcript → analyze flow but for the Construction Services pipeline. Different prompt, different fields, different routing — but identical infrastructure.
